3-lagenmodel

Je hebt 3 lagen op een apperaat. De eerste laag is de fysieke laag dat is de laag waar de hardware bij hoort. Dit is alles wat je kan aanraken dus muis, beeldscherm, cpu enz.. De tweede laag is de logische laag dat is de laag waar je operating system op staat dus bijvoorbeeld: Windows, Mac, Linux, Android en IOS. De derde laag is de loepassingen laag dit betekend dut elke app op programma op uw computer of smartphone behave camera en apps die standaart op uw operating systeem staan.

informatie cpu

Een programmacode uit te voeren doorloopt een CPU een aantal stappen. Het programma zelf staat in het RAM-geheugen. De processor maakt gebruik van een teller de programmateller die het geheugenadres van de instructie bijhoudt. Het begint met de ophaalfase (fetch-fase) waarin de processor de instructies ophaalt vanuit het ROM geheugen. De code die in het geheugenadres staat wordt gebruikt voor de volgende fase, de decodeerfase. Aan de hand van de code in het geheugenadres worden de verschillende componenten van de processor in de juiste stand gezet om de bewerking uit te voeren. Als de processor in de juiste stand staat gaan we over naar de uitvoerfase (execute-fase) waarin de berekening wordt gestart. Als de berekening voltooid is volgt de opslag-fase (stor-fase) waarin de uitkomst van de berekening in het opgeslagen wordt.
